Not sure if you ended up making the session or not, but I know these questions didn’t get answered. I think the best answer is to use whatever you’re comfortable with. I choose to use Flash/Flex and raw Actionscript 3 because they are the standard, and that works well when handing off projects and sharing between multiple developers. I feel that Flex does a very good job out of the box offering nice-looking UI components that are skin-able, and the development environment of Flex builder or Eclipse is coder friendly, which makes it easy to dive into heavy Actionscript when you need to. The site mentioned above, http://www.digitaldollhouse.com was build using Flex builder, but the main use of the Flex framework was for buttons and standard UI components, while the bulk of the application was custom Actionscript.

Hope that helps,
Scott
